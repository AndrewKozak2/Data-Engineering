Лабораторна робота №3
Етапи виконання:

1. Створити python або java проєкт який буде відігравати роль продюсера.
2. Продюсер читатиме з csv файлу дані й на базі кожного запису формувати
   повідомлення/подію яку відправлятиме до Kafka кластеру.
3. Kafka кластер має складатися з наступних компонентів:
   a. Zookeeper
   b. Broker1
   c. Broker2
   d. Schema registry(Опціонально, у випадку використання Avro для серіалізації
   повідомлень)
   e. Kafka UI
4. Kafka кластер повинен мати два топіки (Topic1, Topic2), кожне сформоване повідомлення
   продюсером потрібно публікувати у Topic1 і Topic2.
5. За допомогою Kafka UI перевірте чи надходять повідомлення.
6. Продюсер, Kafka кластер повинні запускатися за допомогою Docker.
7. Для проєкту створити Git репозитарій. За комітати всі зміни й відправити(push) до
   віддаленого Git репозитарію.
8. Посилання на віддалений гіт репозитарій прикріпити до відповідної лабораторної
   роботи у Moodl.

Лабораторна робота №4
Етапи виконання:

1. Створити Java проєкт, Spring, Quarkus або будь-який інший Java фреймворк.
2. До проєкту додати залежності для роботи Kafka Streams
3. Підписатися на один з топіків з попередньої лабораторної роботи й прорахувати
   наступні моменти(агрегуйте повідомлення по даті поїздки):
   a. Яка «середня» тривалість поїздки на день?
   b. Скільки поїздок було здійснено кожного дня?
   c. Яка була найпопулярніша початкова станція для кожного дня?
   d. Які станції входять у трійку лідерів станцій для поїздок кожного дня(враховувати
   початок і кінець поїздки)?
4. Розв’язок кожного обчислення відправити назад у Kafka в окремий топік.
5. За комітати всі зміни й відправити(push) до віддаленого Git репозитарію.
6. Посилання на віддалений гіт репозитарій прикріпити до відповідної лабораторної
   роботи у Moodl.
